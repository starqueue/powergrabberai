One Codebase, Two Manifests

Shared Code

background.js and popup/ are identical across all browsers. The api shim detects browser vs chrome namespace automatically.

Chrome Manifest

manifest.json — MV3 format. Uses service_worker for background, action for popup, host_permissions for URL access.

Firefox Manifest

manifest.firefox.json — MV2 format. Uses scripts array for background, browser_action for popup, flat permissions array.

Manifest Comparison

manifest.json (Chrome/Edge/Safari)
"manifest_version": 3
"background": {
  "service_worker": "background.js"
}
"action": { ... }
"host_permissions": ["<all_urls>"]
manifest.firefox.json (Firefox)
"manifest_version": 2
"background": {
  "scripts": ["background.js"]
}
"browser_action": { ... }
"permissions": [..., "<all_urls>"]

Extension Capabilities

✓

5 webRequest Event Hooks

onBeforeRequest, onSendHeaders, onHeadersReceived, onCompleted, onErrorOccurred

✓

Request/Header Merging

Pending request map combines onBeforeRequest + onSendHeaders with 500ms periodic flush

✓

Tab URL Resolution

Bounded LRU-style tab cache (500 entries) maps tab IDs to URLs for context

✓

Offline Buffering

1000-event buffer when WebSocket is disconnected. Events sent on reconnect.

✓

Self-Traffic Filtering

Extension ignores its own WebSocket traffic to prevent capture loops

✓

Browser Detection

userAgentData API with fallback UA sniffing. Identifies Chrome, Firefox, Edge, Safari, Opera, Brave

✓

Configurable Backend

Backend URL and auth token stored in chrome.storage.local, editable from popup

✓

Exponential Backoff

WebSocket reconnection from 1s to 30s max. Automatic recovery on disconnect.
