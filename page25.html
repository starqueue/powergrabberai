Personal vs. Corporate Account Detection

PowerGrabber AI uses multiple heuristics to flag when a personal AI account is being used for corporate work — a compliance risk hiding in plain sight.

Org ID Mismatch

Keys tied to a different organization ID than the corporate account on file. The sk-proj-x... key routes to org-7Kq... which does not match your corporate org-2Xm...

Detected: Different org ID → likely personal account

Billing Pattern Analysis

API keys on per-token billing that don't appear in corporate cost allocation dashboards suggest shadow spending on personal credit cards.

Detected: Key not in corporate billing → personal spend

Usage Time Correlation

Keys used exclusively outside business hours or on weekends may indicate personal accounts being used for side projects — or after-hours corporate data exfiltration.

Detected: 91% weekend usage → flagged for review

Cross-tool Key Sharing

The same key appearing in multiple unrelated tools (CRM + coding IDE) suggests it was manually copied — not provisioned through your secrets manager.

Detected: Key in 2+ tools without vault origin
