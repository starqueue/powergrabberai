What We Protect

8 Header Types

Sensitive Headers

Authorization, Cookie, Set-Cookie, Proxy-Authorization, X-API-Key, WWW-Authenticate, X-CSRF-Token, X-Auth-Token. Case-insensitive matching. Values replaced, keys preserved.

14 URL Patterns

Query Parameters

token, access_token, api_key, apikey, key, secret, password, session, session_id, auth, code, client_secret, refresh_token, id_token. Sanitized at parse time.

Defence in Depth

Layered Protection

Redaction happens at the protocol parsing layer — the earliest possible point. Sensitive data never enters the application pipeline. Even Debug formatting is sanitized.
