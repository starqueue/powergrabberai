Attribution Rules (TOML Config)

Rules match the active browser tab URL and AI tool to a team. First match wins.
# Team attribution rules in powergrabber.toml

[[team_rules]]
name = "Engineering"
tab_url_contains = ["github.com", "gitlab.com", "linear.app/eng"]
tools = ["cursor", "copilot", "claude-code"]

[[team_rules]]
name = "Data Science"
tab_url_contains = ["jupyter", "colab.research.google.com", "kaggle.com"]
tools = ["chatgpt", "claude"]

[[team_rules]]
name = "Marketing"
tab_url_contains = ["hubspot.com", "figma.com/marketing", "canva.com"]
tools = ["chatgpt", "claude", "jasper"]

[[team_rules]]
name = "Product"
tab_url_contains = ["linear.app/product", "notion.so", "miro.com"]
tools = ["chatgpt", "claude"]

Explicit Team Selection

The extension popup provides a team dropdown that overrides automatic rules. This is useful when:

  • A user works across multiple departments
  • The tab URL doesn't match any rule (e.g., using ChatGPT from a generic browser tab)
  • A contractor is temporarily assigned to a different team
  • The user wants to attribute a session to a specific project or cost centre

The selected team persists until the user changes it. All API calls made while a team is selected are attributed to that team, regardless of tab URL matching rules.
