Denial of Wallet as an OWASP Risk

LLM10

What Is Denial of Wallet?

Unlike traditional denial-of-service attacks that exhaust compute resources, Denial of Wallet attacks exhaust financial resources. Pay-per-token AI APIs create a direct link between uncontrolled consumption and uncontrolled cost. A single runaway loop can burn through an entire monthly budget in hours.

LLM10

Why Existing Tools Miss It

Traditional API gateways monitor request counts. But AI API costs are driven by tokens, not requests. A single request can consume 128,000 tokens. Rate-limit headers are the only real-time signal for token-level consumption — and most security tools ignore them entirely.

Detection

Header-Based Visibility

Every OpenAI, Anthropic, and Google response includes rate-limit headers showing token consumption, remaining quota, and reset windows. PowerGrabber reads these headers on every response to build a real-time consumption graph — no body inspection needed.

Prevention

Alert Before the Bill Arrives

PowerGrabber fires alerts when consumption exceeds baseline thresholds. The +340% spike in this incident was detectable within 17 minutes. Without header monitoring, this incident would only be visible on next month's invoice.
